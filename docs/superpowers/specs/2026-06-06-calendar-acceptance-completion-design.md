# 日历日程验收补齐设计

## 目标

按用户提供的日历日程功能清单补齐完整页面链路。验收以真实浏览器操作为准：页面点击、保存成功、刷新或搜索可回查，必要时用 API/数据库确认子表落库。

## 范围

- 日程创建：标题、起止时间、跨天、全天、时区、地点/线上链接、备注、标签、提醒、重复、参与人、部门、外部联系人、附件、待办。
- 日程协同：忙闲查询、回执、发起人统计、发起人与授权编辑者权限。
- 日程维护：编辑并通知、普通删除、重复日程单次/全系列删除。
- 查阅与导出：搜索、日历筛选、管理员/用户导出 Excel。

## 方案

沿用现有 Vue 表单、Spring Boot API、PostgreSQL 表结构。优先修复阻塞验收的链路：

1. 新建日程附件先在前端暂存，日程保存返回 ID 后批量上传并关联。
2. 日历首页保存后必须能找到刚保存的日程，超过单日展示上限时显示更多入口或切换到搜索可见。
3. 重复日程实例保留原系列 ID 和原始开始时间，单次编辑/删除写入 `event_exception`，全系列操作更新主事件。
4. 外部联系人创建使用数据库序列稳定生成 ID，回执和通知保持可回查。
5. E2E 用例按真实验收路径执行，不再以“控件存在”作为功能通过。

## 验收

- 页面保存普通日程后刷新可见，搜索页可查。
- 页面新建日程时上传附件，保存后详情页显示附件。
- 页面保存重复日程后能看到展开实例，单次编辑/删除不影响全系列。
- 参会人回执能更新状态，发起人能看到统计。
- 导出按钮触发真实 `.xlsx` 下载。
